# 智能学术助手系统设计文档

## 1. 系统架构

### 1.1 整体架构

```
+-----------------------------------+
|        前端应用        |
| (React + Vite)        |
+-----------------------+
            |
+-----------------------+
|        后端服务        |
|   (FastAPI + Python)  |
+-----------------------+
            |
+-----------------------+       +-----------------------------------+
|  核心功能模块          |       |  外部服务集成          |
|  - 论文润色模块        |       |  - 大模型 API         |
|  - 文献分析模块        |       |  - DeepSeek API       |
+-----------------------+       +-----------------------------------+
            |
+-----------------------+
|       数据存储        |
| (localStorage + 文件系统) |
+-----------------------+
```

### 1.2 模块划分

1. **前端模块**：
   - 用户交互界面
   - 文本输入/文件上传功能
   - 润色结果对比展示
   - 文献重点标记功能

2. **后端模块**：
   - API接口
   - 核心功能处理
   - 文件解析
   - 数据处理

3. **核心功能模块**：
   - 论文润色模块：基于提示词模板，支持中英文
   - 文献分析模块：提取研究方法、结果与结论
   - 内容解释模块：提供学术术语解释

4. **数据存储**：
   - 用户设置（API密钥等）
   - 上传文件
   - 临时数据

## 3. 核心功能设计

### 3.1 论文文本润色

- **支持语言**：中文、英文
- **润色类型**：
  - 表达润色
  - 逻辑检查（**新增严格逻辑矛盾检测**）
  - 去AI味
  - 翻译（中文→英文，英文→中文）
  - 缩写/扩写（**新增中文支持**）
- **实现方式**：基于提示词模板，通过大模型API进行处理

### 3.2 文献数据解析与关键信息提取

- **支持格式**：PDF、DOCX
- **提取内容**：
  - 论文标题
  - 作者信息
  - 摘要
  - 研究方法
  - 研究结果
  - 结论
  - 关键词
  - 生成大纲
- **实现方式**：
  - 文件解析 → 文本提取 → AI分析 → 结构化输出

### 3.3 内容解释与背景知识补充

- **功能**：
  - 术语解释
  - 背景知识补充
  - 内容深度解析
- **实现方式**：
  - 内容识别 → AI分析 → 知识解释

### 3.4 用户交互流程

- **文本输入**：支持直接输入文本
- **文件上传**：支持PDF、DOCX格式
- **结果展示**：
  - 润色前后对比
  - 文献关键信息展示
  - 内容解释结果

## 4. 系统流程

### 4.1 论文润色流程

1. 用户输入文本或上传文件
2. 系统提取文本内容
3. 用户选择润色类型和语言
4. 系统调用大模型API处理
5. 显示润色结果与对比
6. 用户可以查看修改日志、复制或重新润色

### 4.2 文献分析流程

1. 用户上传文献文件
2. 系统解析文件并提取文本
3. 系统分析文本结构
4. 提取关键信息（标题、作者、摘要、研究方法、结果、结论、关键词）
5. 生成文献大纲
6. 显示分析结果

### 4.3 内容解释流程

1. 用户输入需要解释的内容
2. 系统调用大模型API分析
3. 生成详细解释和背景知识
4. 显示解释结果
5. 用户可以查看和复制解释内容

## 6. 系统实现要点

1. **文件解析**：
   - PDF解析使用PyPDF2提取文本
   - DOCX解析使用python-docx提取文本
   - 大文件处理采用临时文件存储方式

2. **AI模型调用**：
   - 动态加载提示词，支持中英文提示词
   - 灵活配置API参数，支持不同模型
   - 错误处理和重试机制

3. **用户界面**：
   - 实时反馈，显示处理状态
   - 响应式设计，适配不同设备
   - 简洁美观的交互体验

4. **性能优化**：
   - 异步处理，避免阻塞界面
   - 缓存机制，提高重复请求速度
   - 资源合理利用

## 7. 部署说明

### 7.1 本地部署

- **环境要求**：
  - Python 3.10+
  - Node.js 14+
- **部署步骤**：
  1. 克隆项目
  2. 安装后端依赖：`pip install -r requirements.txt`
  3. 安装前端依赖：`cd frontend && npm install`
  4. 启动后端服务：`uvicorn app.main:app --reload`
  5. 启动前端服务：`npm run dev`

### 7.2 一键启动

- **使用启动脚本**：
  - 双击运行 `start_app.bat` 文件
  - 等待服务启动完成
  - 在浏览器中访问前端服务地址

### 7.3 生产部署

- **容器化**：使用Docker
- **云平台**：可部署到AWS、Azure、GCP
- **CI/CD**：配置自动化部署流程

本系统设计采用前后端分离架构，结合AI大模型能力为学术研究提供了一个功能强大的智能助手。系统能够通过模块化设计和灵活的API调用，实现论文润色、文献分析和内容解释等核心功能，帮助用户提高学术研究效率和质量。

最近的更新包括：

通过不断优化和扩展，系统有望成为学术研究领域的重要工具，为用户提供更加智能、高效的学术辅助服务。
